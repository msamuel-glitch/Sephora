Three systems.
One test.

We locked away October–December data. Built everything on January–September only. Then asked three systems to predict what customers would buy. P@3 = of every 3 recommendations made, how many correct.
Popularity Engine — the floor
4.02%

Recommends the most-purchased brands for everyone. Any intelligent system must beat this.

Hybrid Engine — discovery layer
0.52%

Lower raw accuracy by design. Built for discovery, not prediction. Surfaces brands the ML model would never recommend.

ML Model — the predictor
5.26%

31% more accurate than the popularity baseline. 1.24 percentage points above the floor.

Four things we do not yet do

Sequential journeys — Horizon 2We recommend one brand per occasion. Next: which sequence over three visits maximizes customer lifetime value.
Consideration set — Horizon 2Two recommended brands may compete for the same occasion. Next: occasion-level routing.
Lapsed customers — Horizon 3Our model learns from active customers. High-value lapsed customers are not yet in scope.
Seasonal regime — Horizon 2The model was trained on January–September standard purchase behavior. December gifting mechanics represent a distinct behavioral pattern requiring a seasonal model variant.

We did not build a recommendation engine. We built a system that finds the moment — before it happens — when a customer is about to discover a brand she has never tried. And we made sure nothing gets in the way of that moment.
